# 저에 관하여

반갑습니다.  
이곳에서는 제가 어떤 사람인지 무엇을 좋아하고 어떤 고민을 하며 코드를 쓰는지 소개해 보려 합니다.
글솜씨가 부족해 표현이 다소 어색할 수 있지만, 저라는 사람을 솔직하게 담아내려 노력했습니다.

### 저는
컴퓨터를 좋아하는 사람입니다.  
제가 속한 커뮤니티에서 오신 분들도 많으리라 생각합니다.

---

## 흥미
호기심이 많았던 저는 여러 전자기기의 내부를 관찰하고 분해하며,
무엇이 어떻게 움직이고 작동하는지 파악하며 개조하는 과정에서 즐거움을 느꼈습니다.
중학교 무렵에는 우연히 프로그래밍이라는 세계를 접하게 되었고,
창작의 즐거움과 때로는 그 고통까지 함께 배우게 되었습니다.
그러면서 진로는 프로그래머로 향하게 되었습니다.
프로그래밍이 즐거운 이유는 하고 싶은 아이디어를 몇 가지 명령어로 직접 구현하는 과정에서
만족과 재미가 동반되기 때문입니다.

지금 보고 계신 이 페이지 역시 명령어의 결과물입니다.

---

## 프로그래밍
무언가를 만드는 것에 흥미가 있다면 프로그래밍을 시도해 보길 권합니다.
처음 접하는 코드는 부담스럽게 느껴질 수 있지만,
조금씩 알아가다 보면 특별한 결과물이 만들어지는 과정이 생각보다 흥미롭다는 것을 알게 되실 겁니다.

*(하단부터는 프로그래밍 주제의 글들 위주로 구성되어 있습니다.)*

입문을 고민하고 계신다면, 여러 가지 언어 중에서 [Python](https://www.python.org/)이
다른 언어에 비해 비교적 쉽게 배우고 사용할 수 있어 추천합니다.
프로그래밍을 처음 배울 때 사용했던 언어이기도 하며, 복잡한 초기 설정이 필요하지 않습니다.
```python
print("Keep it simple! stupid.")
```
글을 써 내려가듯 자연스러운 문법이 특징입니다.

### 좋아하는 프로그래밍 언어
저수준 프로그래밍 언어를 좋아합니다.  
배우기 어렵더라도, 섬세한 제어와 속도를 얻을 수 있습니다.
그러나 속도는 개발자의 역량에 따라 달라지는 영역이기에,
항상 코드를 신중하게 작성하려고 노력합니다.

- [Rust](https://www.rust-lang.org/)
```rust
fn main() {
    println!("Compiler is your bf.");
}
```
제가 써본 언어 중 런타임 오류가 비교적 적고 안전했습니다.
C/C++의 속도를 사랑하지만, 그보다 '안전'이라는 가치가 절실할 때 주저 없이 선택하게 되는 언어이기도 합니다.
메모리 안전을 철저히 기하는 특성상 C/C++만큼의 극단적인 저수준 최적화는 조금 까다로울지 모릅니다.
하지만 GC(Garbage Collector) 없이도 소유권(Ownership)이라는 독보적인 시스템을 통해 메모리를 관리하는 점은 특이합니다.
대부분의 개발자들이 이 시스템 때문에 골머리를 앓습니다. 저 또한 그러한 재미를 피할 순 없었습니다.

Rust는 개인 디스코드 봇인 [Rustoki](https://github.com/Kkitut/Rustoki)에 사용하고 있으며,
그 외의 봇 역시 Rust로 만들고 있습니다. (라이브러리는 [Serenity](https://github.com/serenity-rs/serenity)를 사용합니다.)
서버 백엔드 역시 Rust로 구현했습니다. 안전성이 중요한 백엔드와의 궁합이 좋았습니다.

Rust를 이야기할 때 빼놓을 수 없는 요소가 Cargo입니다.
빌드나 라이브러리의 추가가 간편한 매력이 있습니다.

빛이 강하면 그림자도 깊듯이, Rust의 강력한 안전성 뒤에는 기나긴 컴파일 시간이라는 대가가 따릅니다.
첫 릴리즈 빌드 시 느껴지는 5분의 정적은 개발자에게 인내심을 요구하곤 합니다.
다행히 캐싱을 통한 재빌드는 조금이나마 수월하지만,
다른 언어의 빠른 피드백 속도와 비교하면 여전히 고단한 과정임은 분명합니다.
어쩌면 그 컴파일 시간은 잠시 숨을 고를 여유를 주는 순간이 아닐까요.
잠시 생각을 가다듬거나, 스트레칭을 하며 시간을 보내보도록 합시다.

- [C](https://www.c-language.org/)
```c
#include <stdio.h>
int main(void) {
    printf("Grandfather");
    return 0;
}
```
Rust를 알기 전 가장 먼저 손에 익힌 언어입니다.

흔히 입문용 언어로 널리 추천되곤 하지만, 개인적으로는 그 선택에 의구심을 가졌습니다.
많은 이들이 C언어를 어디에 활용해야 할지 갈피를 잡지 못하는 모습을 보아왔기 때문입니다.
단순히 문법을 익히는 것을 넘어 하드웨어에 대한 깊은 이해를 요구하기에
초보자에게 선뜻 추천하기엔 분명 무거운 언어입니다.
가장 중요한 포인터는 신사적인 태도로 신중하고 예의 있게 다루시길 권합니다.

- [C#](https://learn.microsoft.com/ko-kr/dotnet/csharp/)
```cs
public static class Program {
    public static void Main() {
        Console.WriteLine("I hate Unity");
    }
}
```

Java와 높은 상호 유사성을 지닌 C#은 유니티 스크립팅을 통해 입문했습니다.
런타임 기반 언어답게, 어떤 기기에서든 똑같이 작동한다는 점이 장점입니다.
특별히 개성이 있는 것은 아니지만, 그렇다고 흠잡을 곳도 없습니다.
과장되지 않은 담백함은 안정적인 개발 환경과 뚜렷한 결과물을 약속합니다.

### ...
화려한 기술이나 복잡한 최적화보다는 프로그램이 목적에 맞게 안정적으로 동작하는 기본에 집중하는 것을 즐깁니다.
복잡한 알고리즘을 능숙하게 풀어내지는 못하지만,
구상한 설계가 의도대로 구현되는 과정에 매력을 느낍니다.

## 생활

특별한 일이 없으면 온종일 코드 앞에서 시간을 보내곤 합니다.
몰입의 즐거움에 빠져 식사 시간을 놓치기도 하고,
배고픔을 느낀 뒤에야 비로소 자리를 뜨게 되는 불규칙한 생활이 일상이 되었습니다.

디버깅을 하다 보면 어느새 밤낮이 뒤바뀌어 버리는 일도 부지기수입니다.
흐트러진 생체 리듬을 바로잡으려 노력해 보지만,
잡다한 버그를 마주하면 어느덧 모니터 앞에서 밤을 지새우고 마는 저를 발견합니다.
때로는 풀리지 않는 문제 앞에 멍하니 앉아 있기도 하지만,
자리를 비운 그 시간에 번뜩이는 아이디어를 마주하는 기쁨은 이 생활을 지속하게 하는 원동력이 됩니다.

물론 이러한 불균형이 반복되는 삶이 건강에 이롭지 않다는 점은 알고 있습니다.
스스로도 관리의 필요성을 느끼며 규칙적인 휴식과 산책을 섞어보려 시도해 보았지만,
몰입을 이겨내고 실천하는 것이 어렵다는 사실 앞에서 돌아서게 됩니다.
그러나 불균형하더라도 어떻습니까. 인생은 저만의 것입니다.
코드를 써 내려가는 과정 그 자체가 즐겁다면 그것만으로도 충분히 가치 있는 시간이라 믿습니다.
